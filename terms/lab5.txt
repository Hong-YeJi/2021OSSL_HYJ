# id
- 현재 사용자의 실제 id와 유효 사용자 id, 그룹 id를 출력한다.
# chmod
- 파일과 디렉토리의 권한(허가권)을 변경하는 역할을 한다. 파일 모드 변경은 파일 소유자나 슈퍼 유저만 가능하다. 파일 모드는 기호나 8진수로 지정한다. 문자 표기법은 변경할 사용자(대상), 수행할 명령(연산), 설정할 허가권(접근 권한) 세 부분으로 분류된다.
# umask
- 권한을 설정할 때 수동적으로 권한을 주지 않고, 파일이나 디렉토리가 생성될 때 동시에 지정된 권한이 주어지도록 하는 것이다. umask 값에 따라 파일이나 디렉토리의 허가권이 달라지도록 제공하고 있다.
- 새로 만들어진 파일 또는 디렉토리의 권한을 어떻게 설정할 지 제어하는 마스크 설정
- 파일의 기본 허가권 : 666
- 디렉토리의 기본 허가권 : 777
- 파일이나 디렉토리 설정해줄 때마다 매번 허가권을 지정하기 번거로우므로, 디폴트로 허가권 값을 정해놓고 싶을 때 unmask 명령어를 사용한다.
# su
- su 명령어는 Subtitute의 약어로, 일반 사용자 계정으로 로그인한 후, root의 권한을 얻거나 혹은 다른 사용자 계정의 권한을 사용하는 것이 가능하다.
- 정리 : 현재 사용자를 로그아웃 하지 않은 상태에서 다른 사용자의 계정으로 전환하는 명령어
cf) su - 
다른 사용자의 계정으로 완전히 전환하고, 전환한 사용자의 환경설정을 불러오는 명령어
# sudo
- sudo 명령어는 다른 사용자의 보안 권한과 관련된 프로그램을 구동할 수 있게 해준다. 즉, 일반 사용자가 root 권한을 잠시 빌려 명령을 실행하게 하는 명령어이다. Sudo는 권한만 잠시 빌리는 것이기 때문에 근본적으로 명령을 내리는 주체는 현재 사용자이다. Root의 모든 권한을 사용자 계정에게 부여하는 것은 보안상 권장되지 않는다.
- 정리 : 일반 사용자가 root 권한을 잠시 빌려 명령을 실행
# chown
- 파일이나 디렉토리의 소유자를 변경하는 명령어이다.
# chgrp
- 파일이나 디렉토리의 소유 그룹을 변경하는 명령어이다.
